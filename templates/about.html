About Dispatch

Important Disclosure

This page describes how Dispatch is designed to work. Some security claims have limitations. Read the detailed explanations below.

What is Dispatch?

Dispatch is a Tor-only file sharing and communication platform. Users can send encrypted files, chat privately, and manage their accounts. The service is accessible only as a Tor hidden service.

The platform has no advertisements, popups, or sponsored content.

How File Encryption Works (Honest Explanation)

What Actually Happens

  1. Step 1: A random AES-256 encryption key is generated in your browser
  2. Step 2: Your file is encrypted with this key in your browser
  3. Step 3: The encrypted file is uploaded to the server
  4. Step 4: The encryption key is encrypted with the recipient's password hash
  5. Step 5: The encrypted key is stored in the database

Limitation: The server stores an encrypted version of your encryption key. The server owner has access to the encrypted key and the recipient's password hash. With both, the file could theoretically be decrypted.

Mitigation: The password hash is salted and hashed with Argon2id, making decryption computationally expensive. The server does not store plaintext passwords.

Privacy & IP Addresses (Honest Explanation)

What Actually Happens

  • Tor Access: The service is only available as a Tor hidden service. Your real IP address never reaches the server because Tor routes your traffic through the Tor network.
  • IP Address Storage: The server does see and temporarily store the Tor exit node IP address for:
    • Rate limiting (5 requests per minute)
    • Login lockout tracking (5 failed attempts = 30 minute lockout)
    • Security logs (stored for 30 days)
  • No Real IP Exposure: Since all traffic goes through Tor, the server only sees Tor exit node IPs, not your actual location or IP address.

Limitation: The server does log Tor exit node IPs for security purposes. These are stored for 30 days and automatically deleted.

Security Features (What Actually Exists)

Client-Side Encryption

Files are encrypted with AES-256-GCM in your browser before upload. The server receives only encrypted data. The encryption key is encrypted with the recipient's password hash before storage.

Tor-Only Access

The service is exclusively a Tor hidden service. Your real IP address is never exposed to the server. Tor exit node IPs are logged for security purposes.

Two-Factor Authentication

TOTP-based 2FA (Google Authenticator, Authy) with 10 backup codes. Failed 2FA attempts: 5 failures = 15 minute lockout.

Account Lockout

5 failed login attempts = 30 minute lockout. 5 failed 2FA attempts = 15 minute lockout. Lockouts are tracked by username.

Argon2id Password Hashing

All passwords, PINs, and recovery phrases are hashed using Argon2id. This is currently the strongest password hashing algorithm available.

XSS Protection

All user input is HTML-escaped before display. Content Security Policy headers restrict script execution.

HTTP-Only Cookies

Session cookies use HttpOnly, Secure, and SameSite=Lax flags. CSRF tokens protect state-changing requests.

Rate Limiting

Login: 5 per minute. Registration: 5 per hour. File uploads: 20 per hour. Search: 10 per minute.

Session Revocation

Password changes terminate all other active sessions. Banned users are immediately logged out.

Security Headers

Content-Security-Policy, X-Frame-Options, X-Content-Type-Options, Referrer-Policy, Permissions-Policy headers are set.

File Features

  • File Size Limits: Free: 1 GB, Pro: 5 GB, Premium: 10 GB, Owner: unlimited
  • Concurrent Files: Free: 10, Pro: 50, Premium: 100, Owner: unlimited
  • File Expiration: Files auto-delete after 7 days. Pro/Premium users can set custom expiration (1-7 days)
  • Password Protection: Pro/Premium users can add passwords to files
  • File History: View sent/received files with status: pending, accepted, declined, downloaded, expired, cancelled
  • File Type Validation: Only allowed file types can be uploaded (images, documents, archives, videos, audio)

Chat Features

  • Private Chat: Pro and Premium users only. Messages auto-delete after 24 hours.
  • Group Chats: Premium users only. Unlimited members, admin promotion, member removal.
  • Read Receipts: Premium users can disable read receipts. Free and Pro users always send read receipts.
  • Typing Indicators: Real-time typing status for private chats.
  • Message Reactions: Like, Thanks, Agree, Helpful (group chats only).
  • Message Search: Pro+ users can search private chat history.

Account Features

  • User Profiles: View username, role, join date, bio, file statistics.
  • Bio: Pro and Premium users only. Max 200 characters.
  • File Statistics: Files sent/received, storage used, active files.
  • Theme: Light and dark mode. Saved to account.
  • Login History: Last 30 days of logins.
  • Security Logs: Login attempts, 2FA events, password changes, admin actions. Stored for 30 days. IP addresses (Tor exit nodes) are stored.
  • Blocked Users: Block users from contacting you.
  • Account Recovery: 64-character recovery phrase provided at registration. This is hashed with Argon2id.
  • Subscription Expiration: Pro and Premium roles expire after 30 days and revert to Free user.

Admin Features

  • Dashboard: User growth charts, file activity, storage trends.
  • System Health: CPU, memory, disk usage, uptime.
  • User Management: Search, ban, unban, change roles, delete users. Banned users are immediately logged out.
  • Batch Actions: Select multiple users for bulk operations.
  • Subscription Management: Setting a user to Pro/Premium sets a 30-day expiration.
  • Audit Logs: All admin actions are logged with timestamps.

Data Retention

  • Files: Deleted from filesystem and database when expired or manually deleted.
  • Chat Messages: Deleted 24 hours after sending.
  • Login History: Deleted after 30 days.
  • Security Logs: Deleted after 30 days.
  • Failed Login Attempts: Deleted after 7 days.
  • Sessions: Expire after 7 days of inactivity.

Technology Stack

  • Backend: FastAPI (Python) with SQLAlchemy ORM
  • Database: PostgreSQL
  • Encryption: AES-256-GCM (client-side), Argon2id (password hashing)
  • Real-time: WebSockets
  • Authentication: HTTP-only cookies, CSRF tokens, TOTP 2FA
  • Rate Limiting: SlowAPI
  • Frontend: Vanilla JavaScript, HTML5, CSS3
  • Network: Tor hidden service (.onion only)
