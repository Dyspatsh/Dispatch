5. Data Storage & Privacy (Honest Disclosure)

What Data Is Stored

  • Usernames, password hashes (Argon2id), PIN hashes (Argon2id) - stored indefinitely until account deletion
  • Recovery phrase hash (Argon2id) - stored indefinitely until account deletion
  • Encrypted files - stored in filesystem until expiration or deletion
  • Encrypted file keys - stored in database, encrypted with recipient's password hash
  • Chat messages - stored encrypted for 24 hours, then deleted
  • Login history - stored for 30 days, includes Tor exit node IP
  • Security logs - stored for 30 days, includes Tor exit node IP
  • Failed login attempts - stored for 7 days, includes username and Tor exit node IP
  • Session tokens - stored in database, expire after 7 days

What Data Is Not Stored

  • Your real IP address (Tor hides this from the server)
  • Plaintext passwords (only Argon2id hashes)
  • Decrypted file contents (only encrypted files stored)
  • Decrypted messages (only encrypted messages stored)

Data Retention Limits

  • Files: Deleted when expired (7 days default) or manually
  • Chat messages: Deleted 24 hours after sending
  • Login history: Deleted after 30 days
  • Security logs: Deleted after 30 days
  • Failed login attempts: Deleted after 7 days
  • Sessions: Expire after 7 days of inactivity

6. Security (Honest Disclosure)

What Security Measures Exist

  • Argon2id password hashing - computationally expensive, resists brute force
  • AES-256-GCM encryption - files encrypted client-side
  • Tor-only access - real IP never exposed to server
  • Rate limiting - prevents brute force attacks (5 logins/minute, 5 registrations/hour)
  • Account lockout - 5 failed logins = 30 minute lockout, 5 failed 2FA = 15 minute lockout
  • CSRF protection - tokens required for state-changing requests
  • HTTP-only cookies - session tokens not accessible to JavaScript
  • HTML escaping - all user input sanitized
  • Security headers - CSP, X-Frame-Options, etc.
  • Session revocation - password changes terminate other sessions

Security Limitations

  • File encryption keys are stored on the server - encrypted with recipient's password hash, but server owner could theoretically decrypt with access to user password
  • IP addresses (Tor exit nodes) are logged - stored for 30 days for security monitoring
  • Administrators can view encrypted data - though decryption requires the recipient's password
  • No perfect forward secrecy - if server is compromised, stored encrypted keys could be decrypted with user passwords
